## Decisions worth flagging on next iteration

1. **Theme renamed `Money` → `NepMoney`** to avoid collision with `NickFinance.Ledger.Money` value-struct in `@code` blocks. Theme report's brief said `Money`; agent caught the conflict.
2. **`ExportButton` ships CSV, not XLSX.** Plan called for ClosedXML; orchestrator went CSV to avoid the 2 MB dependency. Swap path is one file if real Excel formatting is wanted.
3. **`SitePnL` is a placeholder.** Kernel needs `site_id` on `finance.ledger_event_lines` before the report can be real. Documented in-page and in feature-backlog.
4. **`UserPicker` exposes both `Value` (User entity) and `UserId` (Guid)** — orchestrator-finished pages bind via Guid; if you need the entity, prefer `Value`.
5. **WhatsApp engine hook implementation** — `PolicyApprovalEngine.Plan` fires `IApprovalNotifier.NotifyAsync` when `voucher.AmountRequestedMinor >= NickFinance:WhatsApp:NotifyThresholdMinor` (default GHS 100,000). Phone resolution stub returns null until `IApproverPhoneResolver` has real `user_phones` rows seeded.
6. **Audit log infrastructure complete; no rows yet.** Smoke runner uses `NoopSecurityAuditService` by design. First real CF Access user action will populate `identity.security_audit_log`.
7. **Identity tests gated by `NICKFINANCE_TEST_DB`** — 13/29 cases pass without; remaining 16 need a Postgres test DB. CI integration is a follow-up.
8. **Rate limiter is process-local.** Multi-instance deploy would need Redis-backed limiter; we're single-instance.
9. **EntityPicker has a nullability warning at line 56.** Minor; safe to leave for v1.
10. **TopNav `<UserMenu>` slot still raw.** Sign-out link to `/cdn-cgi/access/logout` could be wrapped in `<AuthorizeView>` to hide it for the dev-fallback user. Small follow-up.

## Operator runbook (post-deploy)

1. Read `docs/PLAN.md` for the architecture rationale.
2. Read `docs/HANDOFF-2026-04-29.md` (this file) for what shipped.
3. To redeploy after a code change:
   ```powershell
   Stop-Service NickFinance_WebApp
   dotnet publish C:\Shared\NSCIM_PRODUCTION\finance\NickFinance.WebApp\NickFinance.WebApp.csproj -c Release -o C:\Shared\NSCIM_PRODUCTION\publish\NickFinance.WebApp
   Start-Service NickFinance_WebApp
   dotnet run --project C:\Shared\NSCIM_PRODUCTION\finance\NickFinance.Database.Bootstrap\NickFinance.Database.Bootstrap.csproj -- --conn "<conn>" --skip-migrations --smoke-test
   ```
4. To onboard a new operator:
   - CF Access dashboard → add their `@nickscan.com` email to the finance app's allow-list
   - First time they log in, a row lands in `identity.users` automatically (status=Active)
   - Grant roles via SQL (or wait for the `/users` admin UI which is a follow-up)
   - Phone number for WhatsApp: insert into `identity.user_phones` (or via the same admin UI)
5. To rotate the DB password:
   - Update Postgres role
   - Update `NICKSCAN_DB_PASSWORD` machine env var
   - Re-run `install-nickfinance-service.ps1` to refresh the connection string env var
   - Restart the service
